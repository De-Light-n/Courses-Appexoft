# 🎓 Як здавати домашні завдання (через Pull Request)

Ми працюємо так само, як у справжніх командах: ви **не пушите напряму в `main`**, а створюєте
**окрему гілку** й відкриваєте **Pull Request (PR)**. Викладач перевіряє його й лишає коментарі.
Зміни потрапляють у `main` **лише після апруву** викладача.

---

## 1. Правило іменування гілок

Кожна гілка має починатися з вашого **скороченого імені**, щоб одразу було видно, чия вона.

**Скорочене ім'я = перша літера імені + прізвище** (латиницею, малими літерами, без пробілів).

> **Приклад (моє ім'я):** Назар Яйко → **`nyaiko`**

Формат гілки:

```
<нік>/hw-m<номер модуля>-l<номер уроку>
```

| Що здаєте | Назва гілки |
|---|---|
| Домашка: Модуль 1, Урок 2 | `nyaiko/hw-m1-l2` |
| Домашка: Модуль 2, Урок 1 | `nyaiko/hw-m2-l1` |
| Виправлення помилки | `nyaiko/fix-opys-korotko` |

> ✏️ Замініть `nyaiko` на **своє** скорочене ім'я. Приклади: Олена Коваль → `okoval`,
> Іван Петренко → `ipetrenko`.

---

## 2. Покроково: від гілки до PR

```bash
# 1. Оновити локальний main
git checkout main
git pull

# 2. Створити свою гілку за правилом іменування (приклад — домашка М1 У2)
git checkout -b nyaiko/hw-m1-l2

# 3. Виконати домашку у відповідному зошиті, зберегти файли

# 4. Додати зміни й закомітити
git add .
git commit -m "Домашка: Модуль 1, Урок 2"

# 5. Відправити свою гілку на GitHub
git push -u origin nyaiko/hw-m1-l2
```

Після `git push` термінал (або GitHub) покаже посилання **Create a pull request** — відкрийте його
й натисніть **Create pull request**.

---

## 3. Що далі

1. Викладач отримає повідомлення про ваш PR (він автоматично призначається рев'юером).
2. Викладач перевірить код і лишить **коментарі** або **апрув**.
3. Якщо є зауваження — виправте, знову `git add` → `git commit` → `git push` **у ту саму гілку**
   (PR оновиться сам).
4. Після **апруву** викладач змерджить PR у `main`.

> ⚠️ **Напряму в `main` пушити не можна** — гілка захищена. Це нормально й правильно:
> так ваш код завжди проходить перевірку.

---

## 4. Часті помилки

| Помилка | Що робити |
|---|---|
| `push` відхилено: `protected branch` | Ви на `main`. Створіть свою гілку (крок 2). |
| Забули оновити `main` перед гілкою | `git checkout main && git pull`, потім нова гілка. |
| Зробили зміни не в тій гілці | `git stash`, перейдіть у потрібну гілку, `git stash pop`. |

Потрібна допомога — питайте викладача 🙂
